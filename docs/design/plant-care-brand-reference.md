## Brand idea

Plant Care should feel simple, calm, personal, and trustworthy. The approved brand concept pairs water and growth: a soft teal droplet containing a small sprout.

The reference sheet uses the tagline:

```txt
Simple care. Healthier plants. Happier you.
```

This line is brand context, not required in the app shell or Home UI.

## Visual cues

Use:

- warm off-white backgrounds
- deep teal for primary identity and main actions
- aqua/teal water tones for watering moments
- soft sage greens for growth, success, and plant context
- restrained dividers and whitespace
- rounded, humanist typography
- simple line icons that echo the rounded brand mark

Avoid:

- loud clinical green
- harsh white dashboard surfaces
- heavy black text blocks
- overdecorated floral styling
- generic task-manager visuals
- diagnosis or health-product cues

## Logo and mark usage

The approved mark is a droplet plus sprout symbol. It can be used sparingly in:

- empty states
- app icon or future app-store contexts
- onboarding or public brand moments

Do not use the brand mark as a replacement for core UI controls. Watering, snooze, reminders, navigation, and plant rows should use accessible icons and text labels.

## Color relationship

The redesign references should remain the implementation source for exact UI roles. The brand sheet supports these directions:

```txt
Primary identity: deep teal
Water accent: aqua / teal
Growth accent: sage green
Background: warm off-white
Surface: soft paper
Text: dark ink, not pure black when possible
```

Urgency colors should stay calm. Overdue watering can use warm terracotta/coral, but should not become alarm-red or diagnostic.

## Typography relationship

The wordmark uses rounded, approachable letterforms. In app UI, prefer a soft humanist sans direction such as Nunito Sans when compatible with the current app setup, with system sans fallback.

Typography should feel readable and practical:

- sentence case over all caps
- moderate weights
- generous line height
- no script fonts
- no proprietary font files committed to the repo

## Product guardrails

The brand should reinforce the watering-first product loop. It must not shift the app toward:

- a plant encyclopedia
- a generic household task app
- a health diagnosis product
- a calendar-first scheduler
- AI-authoritative care guidance
